Bash is (more or less) the language used by the terminal to do things. My bash knowledge is basically zero. I can make folders, change directories, maybe do a couple of git commands, and that’s it.
Every once in a while I learn how to do something cool in bash - something which would have been really tedious to do another way. The problem is that, after I’ve made bash do something cool, I then immediately forget how to do it. There have been loads of times where I see a problem, go ‘oh! this is real easy to do in bash’, then I stare blankly at a terminal screen for 5 minutes and eventually do something else instead.
This post is just a collection of bash code I’ve used at some point, maybe with a short explanation of what the code does. I’m hoping that this will eventually become a sort of personalised cheat-sheet for me. Who knows, maybe I’ll actually start to remember some of this stuff…

Making lots of things at once

Problem: You want to make lots of folders with the pattern name_01, name_02, …, name_10.

Solution: This can be done using curly brackets in one line, which is nice because it saves you from needing to do a loop. Here’s the code

mkdir name_{01..10}

Make sure you remove the folders before trying to make them, otherwise bash will complain. You can get rid of them by doing rm -rf name_*. This will delete every folder with a name name_<something>. It won’t check before deleting them so be careful and check before you run this!

The curly bracket is making an array, and bash is automatically looping over it. If you wanted to do the same thing using loops, this would work

for i in {01..10}; do mkdir name_$i; done

Bash needs the semicolons in those positions - otherwise it won’t work.

You can keep adding more commands if you want, the important thing is to separate them with a semicolon - otherwise bash will try to interpret the extra commands as arguments to the first command, which can lead to confusing errors. For example if you want to add a file to each of these new folders you can do this

for i in {01..10}; do mkdir name_$i; touch name_$i/txt_file.txt; done

It’s not possible to do this using the mkdir ... version as far as I know.

Bash scripts

Problem: You think that last command looks daft, and don’t understand why there’s so many semicolons.

Solution: Make a script!

Bash scripts are just files ending in .sh. Scripts let you format your code, which is nice if you’re doing a lot of stuff. First make a file called script.sh in your current working directory, and paste this in

for i in {01..10}
do
  mkdir name_"$i"
  touch name_"$i"/txt_file.txt
done

This is exactly the same as the last command we looked at, but now its more spaced out and a bit easier to read. To run the script just type bash script.sh in the terminal. The only new thing is that variables are now referenced in double quotes like this: "$<variable name>".

Compressing videos

Problem: A video has a very large file size, and you want to make it smaller.

Solution: Use ffmpeg. First you’ll need to download ffmpeg from here, and add ffmpeg to your path so bash can access it. Once everything is set up, go to the directory with the video in and type

ffmpeg -i input.mp4 -vcodec libx265 -crf 28 output.mp4

ffmpeg is a command line tool which processes videos. This will take an input.mp4 video and make a new, compressed version called output.mp4. The 28 sets the level of compression - higher values will make a more compressed video (smaller file size, but lower quality). 28 is a good default number, but try numbers between 24 - 30 and see how the output looks. This example is stolen from this stackexchange question so go there if you want more info.

Sampling a big csv

Problem: You have a huge csv, and you want to make a smaller sample to work with

Solution: Make a new csv which contains the same headers, then write every 10th row of the original csv to the new csv.

Make a ‘big’ csv to practice on

echo "col1,col2,col3" > big.csv
for i in {1..5000}; do echo "a$i,b$i,c$i" >> big.csv; done

This uses arrows > to redirect the text output of a command to a file. By default echo will repeat whatever you type into the terminal, putting > big.csv tells bash to write the output to a file called big.csv instead. The for loop then appends 5000 rows to the csv using >> - if you used > here it would overwrite the contents of big.csv each time.

Just to check this code worked properly, try counting the number of rows

wc -l big.csv

wc is short for word count - it counts the number of words in a file. The -l flag tells it to count the number of lines instead

This includes the header line, so the number is 5001 instead of 5000. You can fix this by removing the header row using tail and then redirecting the output of that command to wc like this

tail -n+2 big.csv | wc -l

head and tail print out the first and last few rows of a file. The first flag lets you ask for a certain number of rows. The -n+2 part in tail tells it to print out everything from row 2 to the end of the file

The pipe | lets you chain function calls, the result is executed left to right (first do tail, then wc).

Now that everything’s set up, you just need to do the steps mentioned in the solution. First write the header out to a new file

head -1 big.csv > small.csv

And write out every 10th row

tail -n+2 big.csv | awk '{if (NR % 10 == 0) print $1}' >> small.csv

awk is a tool for working with csv files. The general syntax is awk '{some_code}' file.csv. The single quotes and curly brackets are needed, otherwise the code won’t run and you’ll get a syntax error.

Finding all files with a certain name pattern

Problem: You have lots of folders & subfolders, and you want to find all files with a certain name in all these folders.

Solution: Use find. As an example, download some monthly police data from police.data.uk. The link should download crime data by month for each police force area in England. There’s a folder for each month, and inside each folder is a bunch of csv’s with crime data for each police force area. I’ve put everything in a folder called police which looks like this

.
|-- police
|   |-- 2019-12
|   |   |2019-12-avon-and-somerset-street.csv
|   |   |2019-12-bedfordshire-street.csv
|   |   |...
|   |...

If you want to find the locations of all the Essex police force data you just do

find police -name "*essex*.csv"

This tells find to look in the police folder for any files with the name like <anything>essex<anything>.csv. Now that you’ve got the files you probably want to do something with them. find has an exec option which lets you execute code on the list of files returned by find. For example you can ask find to print the contents of each csv to the terminal

find police -name "*essex*.csv" -exec cat {} \;

This will run cat on each element of the list. The elements of the list are represented by the placeholder {}, and you need to end the line with a semicolon so bash knows you’re finished typing in the command (the semicolon needs to be escaped with the \ because bash is a bit particular).

As a more realistic example, you might want to combine all the Essex data into a single csv. First make a file containing just the header

head -1 police/2019-12/2019-12-avon-and-somerset-street.csv > essex_crime.csv

The files all have the same header, so it doesn’t matter which csv you get it from. I’ve taken it from Avon & Somerset because it’s the first police force alphabetically

And append the Essex data (ignoring the header row) to this file

find police -name "*essex*.csv" -exec tail -n+2 {} >> essex_crime.csv \;
