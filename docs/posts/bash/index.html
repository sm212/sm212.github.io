Bash is (more or less) the language used by the terminal to do things. My bash knowledge is basically zero. I can make folders, change directories, maybe do a couple of git commands, and that’s it.
Every once in a while I learn how to do something cool in bash - something which would have been really tedious to do another way. The problem is that, after I’ve made bash do something cool, I then immediately forget how to do it. There have been loads of times where I see a problem, go ‘oh! this is real easy to do in bash’, then I stare blankly at a terminal screen for 5 minutes and eventually do something else instead.
This post is just a collection of bash code I’ve used at some point, maybe with a short explanation of what the code does. I’m hoping that this will eventually become a sort of personalised cheat-sheet for me. Who knows, maybe I’ll actually start to remember some of this stuff…

Making lots of things at once

Problem: You want to make lots of folders with the pattern name_01, name_02, …, name_10.

Solution: This can be done using curly brackets in one line, which is nice because it saves you from needing to do a loop. Here’s the code

mkdir name_{01..10}

The curly bracket is making an array, and bash is automatically looping over it. If you wanted to do the same thing using loops, this would work

for i in {01..10}; do mkdir name_$i; done

Bash needs the semicolons in those positions - otherwise it won’t work.

You can keep adding more commands if you want, the important thing is to separate them with a semicolon - otherwise bash will try to interpret the extra commands as arguments to the first command, which can lead to confusing errors. For example if you want to add a file to each of these new folders you can do this

for i in {01..10}; do mkdir name_$i; touch name_$i/txt_file.txt; done

It’s not possible to do this using the mkdir ... version as far as I know.

Bash scripts

Problem: You think that last command looks daft, and don’t understand why there’s so many semicolons.

Solution: Make a script!

Bash scripts are just files ending in .sh. Scripts let you format your code, which is nice if you’re doing a lot of stuff. First make a file called script.sh in your current working directory, and paste this in

for i in {01..10}
do
  mkdir name_"$i"
  touch name_"$i"/txt_file.txt
done

This is exactly the same as the last command we looked at, but now its more spaced out and a bit easier to read. To run the script just type bash script.sh in the terminal. The only new thing is that variables are now referenced in double quotes like this: "$<variable name>".
